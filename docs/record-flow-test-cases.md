# Record Flow Test Cases

このドキュメントは、試合記録画面の手動回帰テストを固定化するためのケース一覧です。

## 前提

- 対象画面: `app/games/[id]/record/page.js`
- ボールカウント: 4ボールで四球
- 2ストライク時のバントファウル: 自動三振

## ケース一覧

### TC-01 四球（4ボール）

1. 打者を開始
2. `ボール` を4回入力
3. 4球目入力後に打者が1塁へ出塁すること
4. カウントが `B=0, S=0` に戻ること

### TC-02 2ストライク時バントファウル

1. `見逃しS` 2回で2ストライク
2. `バントF` を入力
3. 自動で三振（1アウト増加）になること

### TC-03 3アウト自動攻守交代

1. 2アウト状態を作る
2. 3アウト目となる入力（例: 三振）を入れる
3. 自動で次の半イニングに切り替わること
4. パネルが `一球結果` のメイン画面に戻ること

### TC-04 自チーム攻撃/相手攻撃のスコア更新

1. 自チーム攻撃中に得点入力を行う
2. `score_us` のみ増えること
3. 相手攻撃中に得点入力を行う
4. `score_them` のみ増えること

### TC-05 通常代走

1. 走者がいる状態で `通常代走` を開く
2. 対象走者を選択
3. 控え選手を投入
4. 対象塁の走者名が更新されること

### TC-06 テンポラリー

1. 2アウトで投手または捕手が走者の状態を作る
2. `テンポラリー（臨時代走）` が表示されること
3. 実行後、対象塁の走者が置換されること

### TC-07 守備側交代（ポジション選択）

1. 相手攻撃中に `守備側交代` を開く
2. 守備位置を選ぶ
3. 投入先ポジションを選ぶ
4. 控えを投入
5. 選択ポジションに選手が反映されること

### TC-08 1つ戻す（Undo）

1. 何らかの一球入力を確定する
2. `1つ戻す` を押す
3. カウント/アウト/走者/得点が直前状態へ戻ること
4. 対応するDBレコード（pitch/pa/advance）が削除されること

### TC-09 ランナー表示

1. 1塁/2塁/3塁に走者を作る
2. ダイヤモンドとテキストに塁上走者が表示されること
3. テキスト表示が白文字で読み取れること
